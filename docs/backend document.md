# Backend API & Services: Anshin Otasuke Agent

## 1. Core Framework
The backend in [main.py](file:///Users/satyamvatsa/.gemini/antigravity-ide/scratch/anshin-otasuke-agent/main.py) is powered by **FastAPI** running under **Uvicorn**. It hosts API endpoints for state synchronization, simulations, and handles the live LINE Messaging API Webhooks.

---

## 2. In-Memory State Management
For hackathon deployment speed, state is held in-memory via the `AppState` class:
*   `emergency_mode`: Boolean flag toggled during sirens/warnings.
*   `show_overlay`: Controls whether the caregiver dashboard triggers the urgent fullscreen modal.
*   `document_data`: Dictionary caching the details (issuer, amount, summary, etc.) of the last parsed document.
*   `logs`: Queue storing status strings and severity levels (`info`, `success`, `warning`, `danger`) displayed on the dashboard terminal.
*   `route_coordinates` / `route_steps`: Dynamically generated route data mapped from the senior's position to their safe shelter.

---

## 3. Web API Endpoints

### Status Check (`GET /api/status`)
*   **Purpose:** Polled by the frontend dashboard every 2 seconds.
*   **Payload:** Returns the current state keys.
*   **Overlay Resets:** The `show_overlay` flag automatically toggles to `False` once read, ensuring the modal warning only displays once per alert occurrence.

### Demo Simulator (`POST /api/simulate`)
*   **Purpose:** Triggers pre-set testing sequences.
*   **Actions:**
    *   `reset`: Reverts the system state to standard monitoring.
    *   `invoice`: Triggers a document analysis simulation (calls the document agent and generates Google Calendar links).
    *   `siren`: Activates the emergency state, generates evacuation route coordinate arrays and direction logs.

---

## 4. LINE Message Event Handlers

The backend overrides the LINE SDK handlers to support multi-modal events:

| Event Type | API Source Class | Action Logic |
| :--- | :--- | :--- |
| **Text** | `TextMessage` | Detects keywords like `emergency` (activates emergency mode) or `reset`. Default returns a guided menu. |
| **Image** | `ImageMessage` | Downloads document bytes from LINE content servers; processes through Gemini Agent; replies simplified text with due date details. |
| **Audio** | `AudioMessage` | Downloads voice note; analyzes sound context for sirens/warnings; replies with evacuation directions if a warning matches. |
| **Video** | `VideoMessage` | Downloads video stream; evaluates audio track threat levels; replies safety advice. |
| **Location** | `LocationMessage` | Retrieves coordinates; recalculates the Leaflet evacuation map polyline routes. |

---

## 5. Webhook Security & Background Tasks
*   **Endpoint:** `POST /webhook/line`
*   **Signature Verification:** Validates `X-Line-Signature` using `LINE_CHANNEL_SECRET` to prevent request spoofing.
*   **Asynchronous Execution:** Background handlers run via FastAPI's `BackgroundTasks` pool. This ensures that webhook responses are returned instantly to LINE servers to avoid timeouts, while the Gemini network requests process in the background.

## 6. Integration Utilities

### Google Calendar Sync Link
The backend generates direct, pre-filled URL templates for Google Calendar:
```python
def get_google_calendar_url(title: str, deadline: str, summary: str) -> str:
    base_url = "https://calendar.google.com/calendar/render?action=TEMPLATE"
    # Pre-formatted dates for single-click add
    params = {
        "text": f"Anshin Otasuke: {title}",
        "details": f"{summary}\n\nGenerated by Anshin Otasuke AI Agent.",
        "dates": "20260731/20260731"
    }
    return f"{base_url}&{urllib.parse.urlencode(params)}"
```
